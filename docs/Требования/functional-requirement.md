## Функциональные требования

### 1. Управление шарами

Система должна предоставлять пользователю возможность добавлять на игровое поле шары следующих типов:

- **цельный шар** (`type: solid`);
- **пунктирный шар** (`type: dashed`);
- **с отображением расстояния до бортов в шарах** (`type: solidWithCoordinate`).

Пользователь должен иметь возможность свободно перемещать шары по игровому столу с помощью курсора.

При перемещении шары типа `solidWithCoordinate` автоматически меняют отображение расстояния в шарах до ближайшего борта, при этом расстояние всегда должно быть кратно шару.

### 2. Параметры удара

Для выбранного шара пользователь должен иметь возможность задавать параметры удара, включая:

- **винт** (точка попадания по битку);
- **силу удара** (относительная шкала без привязки к физическим единицам).

Для выставления винта система должна показывать модальное окно со схемой, имитирующей биток, с возможностью перемещать по ней точку попадания.

Сила удара выставляется через шкалу из 8 элементов от 1 до 4 с шагом 0.5.

### 3. Линии и траектории

Система должна позволять добавлять линии траектории удара со стрелкой. Линии могут иметь следующие типы:

- сплошная линия (`type: solid`);
- пунктирная линия (`type: dashed`);
- сплошная линия со стрелкой (`type: solidWithArrow`);
- пунктирная линия со стрелкой (`type: dashedWithArrow`).

### 4. Редактирование позиции

Созданная позиция должна быть доступна для дальнейшего редактирования. Пользователь должен иметь возможность:

- удалять элементы позиции;
- копировать позицию;
- повторно открывать и изменять ранее сохранённые позиции.
